# 你缺的不是更聪明的聊天框，而是一张不会丢工作的桌子

现在的 AI 已经很会写代码了。

真正让人累的，是你得一直守着它。

你要记住哪个窗口在修前端，哪个窗口在查后端；要在 Agent 停下来时催一句“继续”；要在换模型后重新解释背景；还要从一大段终端输出里判断它到底改了什么。

开了五个 AI 窗口之后，你没有得到一支团队。

你只是多了五个需要照看的聊天框。

**Chill Vibe 想解决的，就是这件事。**

## 让工作摊开，而不是叠起来

打开 Chill Vibe，看到的不是一个等待提问的空白输入框，而是所有正在推进的工作。

一个项目一列，一个任务一张卡。

前端可以在修交互，后端可以在查恢复逻辑，另一张卡同时跑测试。谁在工作、谁已经完成、谁还等你确认，都留在原来的位置。

你不必翻聊天记录寻找进度，也不必在十个终端之间猜哪个窗口属于哪个任务。

这不是把窗口排得更整齐。

这是把原本只能排队进行的 AI 对话，变成真正可以并行推进的工作。

## 模型可以换，工作不能失忆

Claude 适合梳理方案，Codex 适合落代码。模型各有长处，用户本来就应该随时切换。

但如果每次切换都要从头交代背景，那不叫自由选择，只是换一种方式重新开工。

在 Chill Vibe 里，跨 Provider 切换时，上下文会跟着任务走；切回原来的模型时，原生会话还能继续。任务中断可以恢复，想尝试另一种实现，也能从已有消息直接分叉。

**上下文属于项目，不属于某一个模型。**

## Agent 继续工作，你可以离开电脑

真正有价值的任务，很少在几十秒内结束。

读仓库、修问题、跑测试、重新构建，常常需要十几分钟甚至更久。人不应该为了等一句“完成了”，一直盯着屏幕。

Chill Vibe 可以让长任务继续运行，在 Agent 过早停下时自动追问，并在工作完成后主动回到原卡片汇报。

离开电脑后，扫码打开手机监工，就能查看进度、追加指令、停止任务或切换模型。

午饭前同时安排三件事。回来时，不是重新问三遍“做到哪了”，而是直接检查三份结果。

## 放开权限，不等于放弃控制

Agent 要真正干活，就必须执行命令、修改文件、操作 Git。

这些过程在 Chill Vibe 里会以结构化内容留在聊天中：运行了什么、改了哪些文件、任务推进到哪一步，都能直接看到。

同时，Codex 和 Claude 默认受到独立的危险操作防护。高风险删除和破坏性 Git 操作会在真正执行前被拦截，而不是只靠一句“请小心”提醒模型。

Agent 可以积极工作，但不该悄悄工作，更不该一次失误就伤到工作区之外。

## 少盯窗口，多做判断

Chill Vibe 不想替代你熟悉的编辑器、终端和本地 CLI。

它只做一件事：让多个 AI 真正像一支团队一样工作。

任务有自己的位置，上下文不会轻易丢失，长工作不需要人守着，关键操作始终可见、可控。

你不再负责记住每个 Agent 做到哪里，也不再充当它们之间的人肉消息队列。

你只需要做真正重要的事：判断方向，检查结果，决定下一步。

**Chill Vibe IDE——少盯窗口，多做判断。**
