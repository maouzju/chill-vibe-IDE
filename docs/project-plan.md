# Chill Vibe IDE 项目计划

## 1. 项目目标

在当前目录实现一个本地运行的看板式 AI IDE，核心体验如下：

- 每一列都是一个可独立配置的工作区视角。
- 同一个工作区可以创建多列，方便并行 vibecoding。
- 每张卡片直接就是聊天页面，没有二级页面。
- 列与卡片都支持拖拽重排。
- 卡片高度可以手动拉伸，聊天内容会继续向下延展。
- 每列独立绑定本地 `codex` CLI 或 `claude` CLI。
- 布局、工作区配置和会话上下文会自动持久化。

## 2. 产品边界

### 本期包含

- 看板式多列布局
- 列级工作区设置
- 同工作区多列并行
- 多卡片会话
- 本地 Codex / Claude CLI 对话
- 拖拽排序
- 卡片最小高度拉伸
- 本地状态持久化

### 本期不包含

- VS Code 级别的文件树与编辑器
- 终端面板
- Git 面板
- 插件系统
- 多人协作

## 3. 执行计划

### 阶段 A：文档和边界

- 输出项目计划文档
- 输出设计文档
- 明确数据模型和 CLI 集成方式

### 阶段 B：基础工程

- React + TypeScript + Vite 前端
- Node + Express 后端
- 共享 schema 和状态存储

### 阶段 C：界面与交互

- 看板列渲染
- 列级工作区配置
- 卡片聊天界面
- 拖拽排序
- 卡片拉伸
- 右侧加号新增列
- 去掉顶部工具条

### 阶段 D：CLI 集成

- Codex 适配
- Claude 适配
- SSE 流式输出
- 停止会话与错误回传

### 阶段 E：收口

- 自动保存与恢复
- 开发脚本自动重启
- 类型检查与 lint
- 本地联调验证

## 5. 验收标准

满足以下条件即可视为本期完成：

1. `pnpm dev` 可以拉起前后端开发环境。
2. 每列可独立配置 provider、workspace、model。
3. 同一路径可以创建多列。
4. 每张卡片可直接对话并持续追加消息。
5. 列与卡片支持拖拽排序。
6. 卡片高度可拉伸，聊天内容不会被固定小滚动框困住。
7. 刷新页面后可恢复布局和会话历史。
8. 改动后可通过自动重启脚本重新拉起服务。
