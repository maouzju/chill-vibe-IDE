# Composer 粘贴剪贴板文件 → 以路径形式插入

## 背景

用户在资源管理器复制文件（Ctrl+C）后，在 agent 对话框（composer）里 Ctrl+V，目前什么都不会发生（图片文件会变成图片附件，非图片文件被静默忽略）。用户希望粘贴时把文件的本地绝对路径以文本形式插入输入框，方便直接把路径交给 agent。

## 需求

1. 在 Electron 桌面端，composer textarea 粘贴剪贴板中的**非图片文件**时，把每个文件的绝对路径以文本插入到当前光标位置。
2. 含空格的路径用双引号包裹；多个文件路径用单个空格分隔。
3. 插入点前/后紧邻非空白字符时自动补一个空格，光标落在插入内容末尾。
4. 图片文件（png/jpeg/webp/gif）保持现有行为：进入图片附件，不插入路径。
5. 截图类内存位图（无磁盘路径）不受影响，仍走附件路径。
6. 浏览器（非 Electron）模式拿不到本地路径：不拦截事件，行为与现状一致。
7. 插入后草稿同步（draft persistence）、slash 命令检测、高度自适应与手动输入一致。

## 非目标

- 拖拽文件到 composer（另行处理）。
- 把非图片文件作为附件上传。
