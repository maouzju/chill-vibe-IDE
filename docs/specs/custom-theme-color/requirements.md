# 自定义主题需求

## 背景

当前设置只能切换浅色、深色或跟随系统，强调色固定为蓝色。用户希望新增一个可配置的“自定义”主题：只有选中它时才能调整主题色；浅色、深色、系统三个内置主题保持默认外观，不提供配色入口。

## 用户目标

1. 主题选择为四选一：浅色、深色、系统、自定义。
2. 选中“自定义”后，用户可以设置底色和主题色；底色除了浅/深两个预设外，还提供取色器可选任意颜色，文字明暗模板随所选底色亮度自动切换。
3. 浅色、深色、系统主题下不显示任何配色控件，始终使用默认蓝色。
4. 颜色修改后立即预览，不需要重启应用。
5. 自定义主题（含底色与颜色）随本地状态持久化，重启后继续生效。
6. 用户可以一键恢复默认主题色；切回内置主题时自定义色立即失效但保留记忆，下次切回自定义仍在。

## 功能要求

- 主题按钮组新增“自定义”选项。
- 仅在自定义主题下渲染：底色行（浅/深预设 chip + 底色取色器 + 当前色值）+ “主题色”控件。
- 自定义底色驱动页面背景与面板、菜单、输入框等表面色板整体换肤；点浅/深预设即回到内置表面并清除自定义底色。
- 控件包含颜色选择器、当前颜色值和“恢复默认”操作。
- 内置主题继续使用现有默认蓝色，不改变老用户视觉；动态 accent token 仅在 `theme === 'custom'` 时写入。
- 自定义颜色至少覆盖：主要按钮、选中态、焦点/高亮线、拖放提示、菜单 hover、卡片标题强调和 Git 工具中的主题强调面。
- 用户选取过亮或过暗的颜色时，应用层应在不改变主要色相的前提下自动调整显示色，保证在当前明暗主题中仍可辨识。
- 恢复默认后必须移除动态覆盖，而不是把某个固定蓝色写成新的自定义值。

## 数据与兼容性

- `theme` 枚举新增 `custom`；`AppSettings` 新增 `customThemeBase`（`light`/`dark`，默认 `dark`）、可空的 `customBaseColor` 与可空的 `accentColor`（颜色持久化格式均为小写 `#rrggbb`）。
- 旧状态缺少这些字段时分别归一化为 `dark`、`null` 与 `null`。
- 非法、空白或不完整颜色值归一化为 `null`；非法底色枚举归一化为 `dark`，均不得阻止旧状态恢复。
- 用取色器选底色时，同一 patch 里把 `customThemeBase` 写成该颜色亮度对应的明暗值，主题解析继续只依赖 `customThemeBase`。

## 可访问性与视觉要求

- 填充按钮上的文字应根据主题色自动选择深色或浅色对比色。
- 浅色、深色以及 `system` 解析后的两种主题均需正常显示。
- 控件在桌面和窄视口中不溢出；默认、hover、focus、已自定义和恢复默认状态清晰但不过度装饰。
- 遵守 `docs/ui-principles.md`：复用 token、保持安静、避免新增无意义边框层级。

## 验收标准

1. 浅色/深色/系统主题下看不到配色控件，界面为默认蓝色。
2. 选中“自定义”并选择一个非蓝色颜色后，界面主要强调色立即变化。
3. 在自定义主题内切换底色浅色/深色后，同一自定义颜色仍保持相同色相且清晰可见。
4. 保存并重新加载状态后，自定义主题、底色与颜色仍生效。
5. 点击“恢复默认”后回到当前底色的默认蓝色；切回内置主题后动态覆盖被移除。
6. 非法持久化颜色或底色不会导致应用加载失败。
