# Requirements — Deep session history search

## Goal

“会话历史”搜索必须能找到当前工作区中更早的内部归档，而不是只过滤 renderer 启动时拿到的最近 50 条和每条首尾预览。

## Requirements

1. 空搜索仍展示当前轻量最近历史，不增加启动时的状态体积或 IPC 负担。
2. 输入搜索词后，内部历史必须按需搜索 `session-history/` 中的完整归档，包括已经超出最近 50 条索引上限、但 sidecar 仍然存在的会话。
3. 搜索范围包括标题、session id、provider、model、工作区路径、结束状态和完整消息正文。
4. 搜索只返回当前工作区的轻量摘要；完整 transcript 仍只在用户点击恢复时按 entry id 读取。
5. 结果按归档时间倒序，重复的同一 provider session 只展示最新归档副本，并设置明确的返回上限。
6. 首次深度搜索可以按需建立轻量归档目录；不得在普通启动或普通保存时加载全部历史正文。
7. renderer 必须防止旧请求覆盖新搜索词的结果，并在搜索期间给出明确状态。
8. 恢复搜索结果后，后续深度搜索不应立刻再次展示同一逻辑会话；再次归档该会话后可重新出现。
9. 旧数据没有“已恢复”墓碑，首次目录重建允许把仍在磁盘上的旧归档副本纳入深度搜索；从本版本开始必须持久记录隐藏状态。

## Non-goals

- 不把完整历史正文重新塞回 `state.json`。
- 不在本切片引入 embeddings、SQLite 或常驻全文索引服务。
- 不改变外部 Codex / Claude 历史导入的扫描规则。
