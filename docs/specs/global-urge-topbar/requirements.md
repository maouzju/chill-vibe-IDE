# 全员鞭策（Global Urge）— 需求

## 背景

现有"自动鞭策"（auto-urge）是逐卡开关：每张聊天卡可以在 composer 设置菜单里打开鞭策并选择鞭策类型（profile）。当用户同时开很多 agent 窗口时，逐个开启很繁琐。

## 需求

1. **设置勾选框**：设置面板"自动鞭策"块内新增"全员鞭策按钮"勾选框，**默认不勾选**。
2. **顶部控件**：勾选后，IDE 顶部栏（`app-topbar`）右侧出现：
   - 一个"全员鞭策"开关（打开/关闭全员鞭策）；
   - 一个鞭策类型选择（下拉，选项来自 `settings.autoUrgeProfiles`）。
3. **作用范围**：全员鞭策开关打开时，鞭策被施加到所有**没有自身鞭策设置**（`card.autoUrgeActive === false`）的 agent 聊天窗口：
   - 已自行开启鞭策的卡不受影响，继续用自己的鞭策类型；
   - 工具卡（Git/音乐/白噪音/便签等）不受影响；
   - 被全员鞭策覆盖的卡使用顶部选择的鞭策类型。
4. **行为一致性**：被施加的卡遵循与逐卡鞭策相同的规则：
   - 流结束（streaming → idle）后若最新助手回合不含成功关键词，自动发送鞭策消息；含成功关键词则本轮不再鞭策；
   - 全员开关打开的瞬间，处于 idle 的适用卡立即被鞭策一次（与逐卡开关打开时的即时发送一致）；
   - 依赖"自动鞭策"功能总开关（`autoUrgeEnabled`）：总开关关闭时全员鞭策不生效、顶部控件隐藏。
5. **持久化**：勾选框状态、全员开关状态、类型选择均持久化，旧存档升级时有默认值。

## 非目标

- 不改变逐卡鞭策的任何现有行为。
- 不为"曾被全员鞭策过的卡"引入豁免记忆。
