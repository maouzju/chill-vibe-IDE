# 鞭策本地大模型判定 + Ollama 集成 — 需求

## 背景

自动鞭策目前用"成功关键词"判定 agent 是否完成任务。关键词依赖 agent 配合输出固定词，误判多。

## 需求

1. **判定方式选项**：每个鞭策类型（profile）可以选择判定方式：
   - `关键词匹配`（现状，默认）；
   - `由本地大模型判断`：选择该方式后需要再选用哪个本地模型（来自本机 Ollama 已安装模型）。
2. **判定实现**：会话流结束时，取该回合**最后一段 assistant 文字**发给所选本地模型，要求其**结构化输出**"是否需要继续"（`shouldContinue: boolean`）：
   - `true` → 发送鞭策消息；
   - `false` → 视为完成，本轮不鞭策（逐卡鞭策源则关闭该卡鞭策，与关键词命中一致）；
   - 判定失败（Ollama 不可用 / 输出不可解析 / 未配置模型）→ 保守不鞭策，避免无限催。
3. **Ollama 集成**：IDE 支持调用本地 Ollama（默认 `http://127.0.0.1:11434`，可用 `CHILL_VIBE_OLLAMA_URL` 覆盖）。
4. **一键安装维护**：设置面板提供 Ollama 管理区：
   - 状态显示：未安装 / 已安装未运行 / 运行中（含已装模型列表）；
   - 一键安装 Ollama（Windows 走 winget）；已安装未运行时一键启动服务；
   - 一键安装**按本机内存推荐**的本地模型（qwen3 系列，判定任务轻量，小模型即可）；
   - 安装/拉取过程有状态与日志，可轮询刷新。
5. **ask-user 硬拦截**：若会话最新回合输出了向用户提问的工具（ask-user 消息且尚未被回答），**无论判定方式、无论逐卡还是全员鞭策，一律不发送鞭策**，保持等待用户回答。

## 非目标

- 不做云端模型判定；不做 Ollama 之外的本地推理后端。
- 不改变关键词模式的现有行为（默认仍是关键词）。
