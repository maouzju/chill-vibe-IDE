# 共享工作区改动基线：需求

## 背景

Agent 回合开始时，IDE 会保存 Git 工作区基线，用于在回合结束后生成“文件改动”兜底卡。
旧实现把每个会话看到的未提交文件内容各复制一份；大型未跟踪目录会按并发会话数放大内存。
紧急限额可以阻止内存爆炸，但超限文件会从兜底卡静默消失，用户无法区分“没有修改”和
“修改存在但差异未生成”。

## 必须满足

1. **文件名不静默消失**：只要回合已明确报告某个路径被修改，兜底差异即使因文件大小、
   基线预算或差异预算无法生成，也必须返回该文件名和省略原因。
2. **差异与文件身份分离**：文件路径、改动类型与差异正文采用不同预算；正文可省略，路径不可
   因正文超限而被丢弃。
3. **并发基线共享**：内容相同的基线正文必须通过有界的内容寻址缓存复用，不得让多个同时开始的
   会话在主进程中各持有一份相同字符串。
4. **缓存有硬上限**：共享缓存必须有明确的总字节预算和逐出策略，不能把“每会话泄漏”变成
   “全局缓存泄漏”。
5. **时间点仍独立**：每个回合继续保留自己的逻辑基线；较晚开始的会话不得错误复用较早时间点
   已过期的文件版本。
6. **现有安全上限保留**：单文件差异、详细基线文件数、基线正文总量和最终差异正文总量仍有硬上限。
7. **用户可理解**：差异省略时，文件卡显示简短中英文说明；不渲染空白差异预览。
8. **不误报归属**：回合开始前已脏但未保留正文的文件，不得伪造相对 HEAD 的差异并归因给本回合；
   只显示“基线未保留，无法生成本轮差异”。
9. **兼容旧消息**：没有省略原因的历史 `edits` 消息继续正常解析和显示。

## 不在本次范围

- 把文件系统观察事件精确归因到任意外部进程。
- 保存超大二进制文件正文或为其生成文本差异。
- 自动修改用户仓库的 `.gitignore`。
