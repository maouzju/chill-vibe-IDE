# 便签按工作区记忆 — 需求

## 背景

便签（Sticky Note 工具卡）的内容目前只存在卡片自身的 `card.stickyNote` 字段里。
关闭便签 tab、关闭工作区列后，内容随卡片一起消失。用户希望便签内容能跟着
"工作区"（即列绑定的 `workspacePath`）被记住。

## 需求

1. **按工作区记忆**：便签内容在编辑时按所属列的 `workspacePath` 持久化到一份
   独立于卡片的存档中；卡片或工作区列被关闭不影响该存档。
2. **恢复入口**：在某个工作区重新打开便签（新便签卡内容为空）且该工作区存在
   存档记录时，便签卡内出现明显入口，允许用户：
   - **恢复**：把存档内容填回当前便签；
   - **删除记录**：丢弃该工作区的存档。
3. 用户直接开始输入（忽略入口）时，新的内容照常成为该工作区的最新记忆。
4. 没有绑定工作区路径的列（`workspacePath` 为空）不参与记忆。
5. 旧版本保存的 `state.json` 没有该存档字段，升级后必须能正常加载（默认为空档）。
6. **记住阅读位置**：关闭或切换便签后再打开，恢复上次的滚动位置和光标/选区，不再默认跳回开头。

## 非目标

- 不做多版本历史（每个工作区只保留最新一份内容）。
- 不做跨设备同步。
